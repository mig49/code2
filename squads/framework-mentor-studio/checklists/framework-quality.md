# Framework Quality Checklist

## Purpose
Validate that a signature framework meets minimum quality standards before packaging into an offer.

## Checklist

### Structure (must pass all)
- [ ] Framework has a unique, memorable name
- [ ] Framework has 3-7 steps/components (not fewer, not more)
- [ ] Each step has a clear name, principle, action, and expected outcome
- [ ] Steps follow a logical sequence (or clear relationship)
- [ ] Framework can be represented visually (diagram, canvas, matrix)

### Clarity (must pass 4/5)
- [ ] Can be explained in 2 minutes (elevator test)
- [ ] Someone unfamiliar can understand the core concept
- [ ] No jargon without explanation
- [ ] Each step is actionable (not just theoretical)
- [ ] Clear before/after contrast (what changes with the framework)

### Differentiation (must pass 3/4)
- [ ] Clearly different from existing frameworks in the space
- [ ] Reflects the creator's unique experience/perspective
- [ ] Has specific examples or case studies supporting it
- [ ] Solves a real, named problem for a specific audience

### Teachability (must pass all)
- [ ] Has a teaching guide (2 min, 30 min, and 90-day versions)
- [ ] Has an application tool (canvas, worksheet, template)
- [ ] Has assessment criteria (how to know if it's working)
- [ ] Can be applied independently by a client (not creator-dependent)

### Monetization Readiness (must pass 3/4)
- [ ] Target audience is willing and able to pay for this
- [ ] Framework justifies premium pricing (depth, not surface)
- [ ] Can be delivered in multiple formats (1:1, group, self-paced)
- [ ] Has clear success metrics that demonstrate ROI
